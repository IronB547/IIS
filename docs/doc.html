Implementace

Používáme databázový server (MySQL), HTTP požadavky (GET, POST, PUT, DELETE) obsluhuje server, který je vytvořený pomocí Node.JS. Uživateli zobrazujeme data přes frontendový framework Vue.JS.

Aplikační server využívá Express.JS pro zpracování HTTP požadavků nad databází. Poskytuje přístup ke koncovým bodům pomocí cest (routes). Existují jednotlivé cesty pro: uživatele, tickety a servisní požadavky a jejich detaily. Soubory těchto cest jsou umístěné ve složce „routes“.

Část programu, která komunikuje s databází je umístěná v adresáři „services“. Ta zároveň zpracovává parametry dotazů (query params) a ověřuje správnost vstupů. Autentifikace a autorizace uživatelů probíhá pomocí JWT tokenů. Pro generování tokenů je potřebné, aby ve složce „ .env “ byla nastavená proměnná prostředí TOKEN_SECRET="<secret>". Zároveň je možná i volba portu pomocí PORT="<port>".

Klientská strana je rozdělena do několika pohledů (views) a ty komunikují s aplikačním serverem pomocí metod definovaných v soubourech, které jsou ve složce „stores“. Výše zmíněné JWT tokeny ukládáme v „localStorage“, který zajišťuje perzistenci jednotlivých tokenů. Tyto tokeny odesíláme v HTTP hlavičce každého požadavku.

Rozšíření

  • Jednotlivé tickety jsou propojené s požadavky, které k nim patří.

  • Zobrazování informací o uživatelích.

Instalace

Stručně popište:
  • Postup instalace na server
    • Extrahujeme projekt a zadáme příkaz:
      cd ./frontend/ && yarn install && yarn build && cd ../backend/ && npm install && npm run-script run
  • Softwarové požadavky (verze PHP apod.)
    • node - 16.15.0 a vyšší
    • yarn - 1.22.19 a vyšší
    • npm - 8.11.0 a vyšší
  • jak rozbalit, konfigurovat, inicializovat databázi, ...
    • Používáme GOOGLE cloud hosting pro hostování databáze a pro přístup je nutné povolit staticky IP adresu zařízení. Ve složce „db“ je database.sql skript, který obsahuje celou databázi. Zároveň pro propojení databáze a backendu, je nutné ve složce „backend“ vytvořit soubor „ .env “, do kterého je nutné vepsat:
      DB_PASSWORD="<heslo>"
